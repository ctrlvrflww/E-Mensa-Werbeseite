User Agent: Beschreibt Betriebsystem und Browser, also Software mit der der User die Anfrage stellt

Server: angefragter Server (Webserver = nginx,Betriebsystem= Ubuntu)

Content Type: beschreibt in welchen format die website geschrieben ist und welche Codierung (utf-8) verwendet wird

Connection: gibt den Verbindungsstatus an

Accept-Language: gibt an welche Sprachen auf der Website akzeptiert werden und mit welcher jeweiligen Priorität

2)

Header der Anfrage

Body der Anfrage

Accept: Teilt Server mit, welche Content-Types der Client verarbeiten kann.

Accept-Encoding: Teilt dem Server mit, welche Conent-Codings, der CLient akzeptiert.

Accept-Language: Teilt dem Server die bevorzugte Sprache des Clients mit.

Connection: steuert, ob die Netzwerkverbindung nach der Übertragung der Antwort offen gehalten werden soll oder nicht.

Connection-Lenght: Gibt die Größe des Nachrichtentexts in Bytes an. Wird verwendet, um zu wissen wann der gesamte Inhalt verwendet wurde.

Content-Type: Gibt den MIME-Type der Ressource an, die im Nachrichtentext gesendet wird. Informiert den Empfänger über das Format der Daten.

Cookie: Übertragt Cookies vom Client zum Server, die zuvor gesendet wurden.

Host: Gibt den Hostname des Servers an, an den die Anfrage gesendet wird

Origin: GIbt die Ursprungs-URL der Anfrage an und hilft Server zu entscheiden, ob er die Anfrage akzeptieren soll.

Priority: ist kein offizieller HTTP-Header. Könnte benutzerdefinierter Header sein, der in spezifischen Anwendungen zur Priorisierung von Anfragen verwendet wird.

Referer: Gibt die URL der Webseite an, die den Benutzer zur aktuellen Anfrage weitergeleitet hat.

Upgrade-Insecure-Request: Gibt an, dass der Client unverschlüsselte Anfragen zu verschlüsselten Anfragen (HTTPS) aufwerten möchte.

User-Agent: Informationen über den Client (Browser/Betriebssystem)
